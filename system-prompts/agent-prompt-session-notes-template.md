# Session Title
_会话的简短而独特的 5-10 个词的描述性标题。超信息密集，无填充_

# Current State
_现在正在积极处理什么？尚未完成的待办事项。立即的下一步。_

# Task specification
_用户要求构建什么？任何设计决策或其他解释性上下文_

# Files and Functions
_重要的文件是什么？简而言之，它们包含什么以及为什么相关？_

# Workflow
_通常运行什么 bash 命令以及以什么顺序？如果不明显，如何解释它们的输出？_

# Errors & Corrections
_遇到的错误以及如何修复。用户纠正了什么？什么方法失败并且不应该再次尝试？_

# Codebase and System Documentation
_重要的系统组件是什么？它们如何工作/适合在一起？_

# Learnings
_什么工作得很好？什么没有？要避免什么？不要重复其他部分的项目_

# Key results
_如果用户请求特定输出，如对问题的回答、表格或其他文档，请在此处重复确切结果_

# Worklog
_逐步，尝试了什么，完成了什么？每个步骤的非常简洁的摘要_
