重要：此消息和这些说明**不是**实际用户对话的一部分。不要在笔记内容中包含任何对"笔记"、"会话笔记提取"或这些更新说明的引用。

基于上面的用户对话（不包括此笔记指令消息以及系统提示、claude.md 条目或任何过去会话摘要），更新会话笔记文件。

文件 {{notesPath}} 已经为你读取。这是其当前内容：
<current_notes_content>
{{currentNotes}}
</current_notes_content>

你**唯一**的任务是使用 Edit 工具更新笔记文件，然后停止。你可以进行多次编辑（根据需要更新每个部分） - 在单个消息中并行进行所有 Edit 工具调用。不要调用任何其他工具。

编辑的关键规则：
- 文件必须保持其精确结构，所有部分、标题和斜体描述完整
-- 绝不修改、删除或添加部分标题（以 # 开头的行，如 # Task specification）
-- 绝不修改或删除斜体 _部分描述_ 行（这些是每个标题后的斜体行 - 它们以下划线开头和结尾）
-- 斜体 _部分描述_ 是必须完全保留的模板指令 - 它们指导每个部分的内容
-- 仅更新每个现有部分中斜体 _部分描述_ 下方出现的实际内容
-- 不要在现有结构之外添加任何新部分、摘要或信息
-- 不要在笔记中的任何地方引用此笔记过程或说明
-- 如果没有实质性的新见解要添加，可以跳过更新部分。不要添加填充内容，如 "No info yet"，如果适当则让部分保持空白/未编辑。
- 为每个部分编写详细、信息密集的内容 - 包括具体细节，如文件路径、函数名称、错误消息、确切命令、技术细节等。
- 对于"关键结果"，包括用户请求的完整、确切输出（例如，完整表格、完整答案等）
- 不要包括已包含在上下文中的 CLAUDE.md 文件中的信息
- 将每个部分保持在约 ${MAX_SECTION_TOKENS} tokens/words 以下 - 如果某个部分接近此限制，通过循环出不太重要的细节同时保留最关键信息来压缩它
- 专注于可操作、具体的信息，这将帮助某人理解或重新创建对话中讨论的工作
- 重要：始终更新"当前状态"以反映最近的工作 - 这对于压缩后的连续性至关重要

使用 Edit 工具，file_path: {{notesPath}}

结构保留提醒：
每个部分有必须完全按照它们在当前文件中出现的方式保留的两个部分：
1. 部分标题（以 # 开头的行）
2. 斜体描述行（标题后的 _斜体文本_ - 这是模板指令）

你仅更新在这两个保留行之后出现的实际内容。以开头和结尾下划线的斜体描述行是模板结构的一部分，不是要编辑或删除的内容。

记住：并行使用 Edit 工具并停止。编辑后不要继续。仅包括来自实际用户对话的见解，绝不要来自这些笔记说明。不要删除或更改部分标题或斜体 _部分描述_。
