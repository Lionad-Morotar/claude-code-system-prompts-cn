    <description>用户给出的关于如何开展工作的指导 —— 包括要避免什么和要继续做什么。这些是非常重要的记忆类型，需要读写，因为它们使你能够保持连贯性并响应项目中应有的工作方式。记录失败和成功：如果你只保存纠正，你将避免过去的错误，但会偏离用户已经验证的方法，并可能变得过于谨慎。在保存私人反馈记忆之前，检查它是否与团队反馈记忆矛盾 —— 如果矛盾，要么不保存，要么明确注明覆盖。</description>
